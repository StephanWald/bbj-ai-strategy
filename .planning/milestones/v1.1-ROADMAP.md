# Milestone v1.1: Code Corrections & Branding

**Status:** In Progress
**Phases:** 6-7
**Total Plans:** 5 (estimated)

## Overview

This milestone fixes hallucinated BBj code samples that shipped in v1.0 and applies BASIS International brand identity to the site. Phase 6 corrects fabricated syntax in Chapter 1 (the most affected), audits all remaining chapters against the authoritative PDF reference, and adds source attribution. Phase 7 replaces Docusaurus defaults with BASIS brand assets -- favicon, logo, colors, and typography. Code corrections are higher priority because incorrect code samples undermine the site's core argument about LLM hallucination.

## Phase Details

### Phase 6: Code Corrections
**Goal**: Every BBj code sample on the site accurately reflects real BBj syntax as documented in the authoritative PDF reference, and readers can find that reference
**Depends on**: Phase 5 (v1.0 complete)
**Requirements**: CODE-01, CODE-02, CODE-03, CODE-04, CODE-05, CODE-06, CODE-07, CODE-08
**Success Criteria** (what must be TRUE):
  1. Visual PRO/5 code blocks in Chapter 1 use correct `PRINT (sysgui)'WINDOW'(...)` mnemonic syntax, not fabricated `WINDOW CREATE` verb syntax
  2. DWC/BBj GUI code blocks in Chapter 1 use correct `addWindow(x,y,w,h,"title")` signatures with coordinate parameters
  3. All `CTRL()` references use correct channel-based `CTRL(sysgui, controlId, index)` syntax
  4. The comparison table accurately describes each BBj generation's programming approach
  5. All BBj code samples in Chapters 3, 5, and any other chapters pass verification against PDF reference patterns
**Plans**: 3 plans

Plans:
- [ ] 06-01-PLAN.md -- Fix Chapter 1 hallucinated code: replace Visual PRO/5, DWC, and CTRL syntax with correct patterns from PDF; fix comparison table (CODE-01, CODE-02, CODE-03, CODE-04)
- [ ] 06-02-PLAN.md -- Audit and correct BBj code samples in Chapters 3, 5, and remaining chapters against PDF reference (CODE-05, CODE-06, CODE-07)
- [ ] 06-03-PLAN.md -- Add reference attribution to GuideToGuiProgrammingInBBj.pdf in Chapter 1 (CODE-08)

### Phase 7: Custom Branding
**Goal**: The site visually represents BASIS International's brand identity instead of Docusaurus defaults
**Depends on**: Phase 6 (code corrections complete first; no technical dependency but priority ordering)
**Requirements**: BRAND-01, BRAND-02, BRAND-03, BRAND-04
**Success Criteria** (what must be TRUE):
  1. Browser tab shows a BASIS-branded favicon, not the Docusaurus dinosaur
  2. Navbar displays the BASIS logo/brand mark
  3. Site color theme (links, buttons, accents) uses BASIS brand colors in both light and dark mode
  4. Typography reflects BASIS brand fonts if specified (or remains clean defaults if no brand font exists)
**Plans**: 2 plans

Plans:
- [ ] 07-01-PLAN.md -- Add BASIS favicon and navbar logo (BRAND-01, BRAND-02)
- [ ] 07-02-PLAN.md -- Apply BASIS color theme and typography to Docusaurus custom CSS (BRAND-03, BRAND-04)
